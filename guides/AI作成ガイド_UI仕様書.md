# UI仕様書 作成ガイド

## 目的

本書は、UI仕様書を**AIで安定して作成する**ための記載ルールおよび構成ルールを定義する。  
外部仕様書の画面一覧に記載された画面ベース名を単位として1ファイルを生成する。

---

## 本ガイドの前提


- **画面種別・入力方式は、外部仕様書の「画面一覧」に記載された値をそのまま転記する。AIが独自に判断・変更することは禁止する。**
- マスタ運用およびWF申請対象データの編集可否は、外部仕様書の「データ運用方針」の内容に従う。
- Workflow 画面（画面種別：Workflow）は本ガイドの対象外。`AI作成ガイド_UI仕様書_Workflow.md` を使用すること。
- 帳票（画面種別：帳票）は本ガイドの対象外。`AI作成ガイド_UI仕様書_帳票.md` を使用すること。

### 画面種別と入力方式の定義

| 画面種別 | 概要 |
|---------|------|
| 通常 | 検索＋一覧＋編集フォーム。標準的なマスタ管理画面。 |
| コンテキスト | 業務条件を先に確定（条件設定）→ 一覧表示 → 編集の構成。タブレット等に多い。 |
| 参照 | 検索＋一覧表示のみ。編集不可。 |
| 単一選択 | 入力フォームから呼び出す1件選択ダイアログ。 |
| 複数選択 | 入力フォームから呼び出す複数件選択ダイアログ。 |

| 入力方式 | 概要 |
|---------|------|
| 基本 | 単票入力。新規・更新・削除を行う標準構成。 |
| 親子 | 親フォーム＋明細テーブルのインライン入力。明細は親に完全従属。 |
| 親子選択 | 親フォーム＋既存マスタを検索・選択で紐付ける構成。明細の新規作成は行わない。 |
| 一括 | Excelライクな一括編集テーブル。**一覧画面は作成しない。** |
| 階層構造 | 左ペインにツリー、右ペインに選択ノードの詳細表示・変更・削除フォームを配置する。**一覧画面は作成しない。** |
| - | 選択・参照（入力なし）。 |

### コンテキストの追加前提

- 「条件設定」（親条件を設定する画面）と「一覧表示」（一覧を表示する画面）を分離した構成として記載する。
- 親条件はユーザー単位で保持する。一覧表示には親条件の入力欄を置かない（サマリ表示は可）。
- 一覧内検索は、追加要望がある場合のみ設ける。

---

## 入力資料

UI仕様書の生成前に、以下の資料を**すべて**読み込むこと。どれか1つでも未読のまま生成を開始してはならない。

| 資料 | ファイル名パターン | 主な参照目的 |
|------|--------------------|-------------|
| 業務要件 | `{business name}_業務要件.md` | 業務固有の表示ルール、入力項目の固定値・選択肢、業務フロー上の制約 |
| 業務仕様書 | `{business name}_業務仕様書.md` | 業務ユースケース・画面遷移の概要、データ一覧（主要エンティティと項目）、未確定事項の有無 |
| 外部仕様書 | `{business name}_外部仕様書.md` | 画面一覧（画面ベース名・画面種別・入力方式）、データ運用方针、画面イメージ（Salt） |

> **業務要件に記載された固有ルール（表示カラムの切替、ロール別制限、固定選択肢など）は、外部仕様書・業務仕様書に転記されていない場合でも必ず UI仕様書に反映すること。**

---

## 成果物

- ファイル名：`＜業務名＞_UI仕様書_＜画面ベース名＞.md`
- 出力形式：Markdown
- 出力方法
  - 本文を画面表示せず、必ずファイルとして出力すること
  - 作成した md ファイルは、最後に、人間が見やすいように整形すること

- **サブ画面名**のルール：
  - 一覧と入力フォームの両方がある場合：`## 一覧画面` / `## 入力フォーム画面` のように画面種別に「画面」を付けた名称にする。
  - 一覧のみ・入力フォームのみ・単一構成（階層管理など）の場合：`## ＜画面ベース名＞画面`（例：`## 組織階層管理画面`）とする。
  - コンテキスト・選択画面など複数ステップがある場合：`## 条件設定画面` / `## 一覧表示画面` のように各ステップ名に「画面」を付ける。
- 各サブ画面の `### 項目定義（XXXXX）` はそのサブ画面内の対象ごとに複数設けてよい（例：`項目定義（検索条件）` と `項目定義（一覧）`）。
- `### 画面レイアウト指示` と `### 画面ワイヤー` は**すべての `## ＜xxx＞画面` セクションに必ず設ける**。

## 記載ルール

### 画面定義

- 以下の項目を箇条書きで記載する。名称のみ記載し、説明は不要。
  - 画面ベース名：外部仕様書の画面一覧に記載された画面名をそのまま使用する。末尾に操作種別・UI形態・状態を付与しない。
  - 画面タイトル：実際の画面に表示する名称。ワイヤーの先頭と一致させる。
    - **【コンテキストのみ】** 画面タイトル（条件設定）と 画面タイトル（一覧表示）を個別に記載する。
  - 画面種別：外部仕様書の「画面一覧」に記載された値をそのまま転記する。
  - 入力方式：外部仕様書の「画面一覧」に記載された値をそのまま転記する。
    - 参照・選択ダイアログは記載不要。

---

### 画面概要

- 対象画面の目的を、1〜3行で簡潔に記載する。仕様や動作の説明は記載しない。
- 項目名・UI構造・操作手順は記載しない。
- 外部仕様書の「データ運用方針」の内容に従い、編集権限の制限がある場合は本節に記載する。
- **【選択ダイアログのみ】** 以下を明記すること。
  - 単一選択：「行クリックで選択確定、ダイアログが閉じる。確定ボタンなし。」
  - 複数選択：「チェックボックスで複数選択し、[選択] ボタンで確定する。」

---

### 参照データ定義

- プルダウン・チェックボックス等で使用する選択肢、および一覧で参照表示する他テーブル項目を記載する。
- **本章がデータ定義書（Step 6）の元データとなる。** データ定義書はUI仕様書の本章をベースに自動生成するため、正確に記載すること。
- 固定値とマスタ参照を別で明示する。
- 「取得元」「値」「表示」には**テーブルやカラムの物理名を記載しない**。
- フラグ項目は固定値を作成しない（チェックボックス表記にするため）。
- **【親子選択のみ】** 検索結果一覧（チェックボックス選択対象）は本章に記載しない。
- **【選択ダイアログのみ】** 検索対象マスタ自体の一覧は本章に記載せず「検索仕様ルール」に記載する。
- 存在しない場合は「特になし」と記載する。

**記載例**

```text
参照_区分一覧：
- 取得元：固定値

参照_担当者一覧：
- 取得元：担当者マスタ
- 抽出条件：特になし
- 値：担当者ID
- 表示：氏名
```

---

### 画面レイアウト指示（各 ## ＜xxx＞画面 セクション内）

- ユーザーからのレイアウト要望を箇条書きで自由に記載する。
- **AIはこの章の内容を読んでワイヤーを生成・更新すること。**
- 実装者もこの章を参照してレイアウトを実装すること。
- 内容がない場合は「特になし」と記載する。
- TBD 解消・仕様変更でレイアウト変更が必要になった場合は、本章に追記した後にワイヤーを更新すること。

**記載禁止事項**（他の章に準拠のある内容は記載しない）

| 記載禁止の内容 | 正しい記載先 |
|---|---|
| 画面遷移（ボタン撼下でどこに遷移するか） | `## 操作` 章 |
| ボタンの表示・非表示条件 | `### UI状態切替ルール` |
| 項目の必須・任意・読み取り専用の条件 | `### UI状態切替ルール` |
| 該当する項目が表示される課 | `### UI状態切替ルール` |
| 項目の入力制約（桁数・文字種） | `### 項目定義（XXXXX）` の入力制約/表示仕様列 |
| ソート・絞り込み条件 | `### 検索仕様ルール` |

**記載可能な指示の例**

- 検索エリアのデフォルト表示：展開 / 折りたたみ
- 検索条件の列数：2列
- フォーム列数：2列
- 表示形態：ドロワー
- その他：自由記述

---

### 画面ワイヤー（各 ## ＜xxx＞画面 セクション内）

- 外部仕様書（`＜業務名＞_外部仕様書.md`）の対応する `#### 一覧画面` / `#### 入力フォーム画面` の Salt をそのまま転記する。単一構成画面は `### ＜画面名＞画面` 直下の Salt を転記する。
- ワイヤーの変更・追加が必要な場合のみ、本章内で Salt を修正する。Salt 記法は「外部仕様書ガイド §5.3 入力部品の Salt 記法一覧」に従う。
- **グループ（課など）によって表示項目や構成が異なる場合は、代表的なパターンごとにワイヤーを作成する。** 外部仕様書の代表1パターンをベースに、差異のあるパターンを追加する。各ワイヤーの直前に `**＜グループ名＞**` の見出しを付けて区別する。代表パターン数はUI仕様書策定フェーズで業務仕様書・業務要件を参照して判断する。
- **テーブル内の空セルには全角スペース（`　`）を必ず記入すること**（外部仕様書ガイド § Salt 記法のルール 参照）。

---

### 項目定義テーブルの共通ルール

外部仕様書の Salt タイトルに対応するすべてのサブ画面章で、以下のルールを適用する。

#### 表ヘッダー

| 表示順 | 項目名 | UI部品 | 必須 | 入力制約/表示仕様 |

> 例外：**項目定義（サマリ）** のみ `| 表示順 | 項目名 | 表示仕様 |`（UI部品・必須列なし）

#### 必須列のルール

| 状況 | 必須値 |
|------|--------|
| 入力が必要な項目 | `〇` |
| 任意入力 | `-` |
| 検索条件・表示のみ・チェックボックス・テキスト表示 | `-` |
| 親条件（条件設定セクション） | `〇`（すべて必須） |

#### UI部品と入力制約/表示仕様のルール

- **テキスト入力 / テキストエリア / 数値入力 / 日付入力**：入力制約（桁数・文字種など項目単体の条件）を記載する。なければ `-`。
- **テキスト表示**：表示仕様（加工ルール）を記載する。`初期値：(自動採番)` / `ログインユーザー` など。
- **プルダウン入力**：`参照：＜参照データ名＞` を記載する。
- **プルダウン検索**：選択肢が多くキーワードで絞り込む必要がある場合に使用するプルダウン。プルダウン内のテキスト入力で候補を絞り込める。`参照：＜参照データ名＞` を記載する。
- **チェックボックス**（検索条件）：`デフォルト値：チェックなし` を記載する。
- **ボタン / チェックボックス**（表示列）：`-`。
- 一意性が必要な項目：`重複不可` を追記する。
- 複数項目にまたがる条件は記載せず「項目間ルール（複合チェック）」へ。

#### セクション固有のルール

**〔検索条件〕**
- マスタ管理画面で有効フラグがある場合はチェックボックスを必ず含める。
- プルダウン：他テーブル参照項目 / 区分コードを名称で選択する項目。

**〔入力フォームの項目定義（入力フォーム）〕**
- コードを自動採番する場合：UI部品 `テキスト表示`、入力制約/表示仕様 `初期値：(自動採番)`。採番ルールは「操作」章に記載する。
- 操作者記録項目（記入者・作成者など）は テキスト表示・ログインユーザー で仮設計し、TBD とする。
- 編集対象テーブルの主キー（サロゲートキー）は記載対象外。
- ワークフロー関連項目（件名・申請日・申請者・承認者・承認経路・WF案件ID）は記載対象外。
- **【親子のみ】** 項目定義（＜明細名称＞）を明細種類ごとに追加する。
- **【親子選択のみ】** 項目定義（＜子名称＞ 検索条件）と項目定義（＜子名称＞）を追加する。親フォームがない場合は項目定義（＜親名称＞）を省略する。
- **【複数選択ダイアログのみ】** 項目定義（選択一覧）の先頭行に「選択（チェックボックス）」を追加する。
- **【階層構造のみ】** 項目定義（検索条件）を設ける（ツリーの各階層レベルに対応するプルダウンで絞り込む）。項目定義（詳細）を設ける（選択ノードの表示・変更対象項目）。項目定義（追加）を別途設ける（新規ノード作成時の入力項目）。

**〔一覧編集の項目定義（一覧編集）〕**
- 入力フォームと同じルール。実行ボタンは記載対象外。

#### 検索仕様ルール

- 取得対象外条件・ソート順のみ記載する。以下はデフォルト仕様として記載対象外。

| 部品 | デフォルト（記載不要） |
|------|----------------------|
| 一覧 | 文字列：部分一致 / 最大10件 / ページングあり |
| 一覧表示（コンテキスト） | 同上 |
| 親子選択の子 | 文字列：部分一致 / 全件表示 / 最大100件 / スクロール |
| 選択ダイアログ | 文字列：部分一致 / 全件表示 / クライアントサイドフィルタ / 最大100件 / コード昇順 |

#### 項目間ルール（複合チェック）

- 複数項目にまたがる業務上の成立条件のみを記載する。単項目チェックは入力制約/表示仕様へ。
- 存在しない場合は「特になし」と記載する。

#### UI状態切替ルール

- ロールや状態による表示可否・操作可否の条件のみを記載する。
- 入力フォームでは新規モード／更新モードの差分（編集不可項目など）も記載する。
- 存在しない場合は「特になし」と記載する。

#### 出力定義（Excel／CSV）

- ファイルダウンロード機能がある `### 項目定義（XXXXX）` の直下に `### 出力定義（Excel）` または `### 出力定義（CSV）` を設ける。
- 以下の内容を記載する。
  - 出力単位（1明細 = 1行、など）
  - 出力対象の絞り込み条件（検索結果全件、など）
  - 出力列定義を以下の列構成の Markdown 表で記載する。

| 出力順 | 列名 | 出力内容 |

- 出力内容には、表示仕様（加工ルール・書式）を記載する。
- 列名および出力内容には**テーブルやカラムの物理名を記載しない**。

---

### 操作

- 以下の共通操作は記載しない。

  | 区分 | 記載しない共通操作 |
  |------|--------------------|
  | 通常・コンテキスト・参照 | [検索][クリア]、[選択]ボタンの一覧→ダイアログ表示 |
  | 基本 | [登録][更新][削除]、ダイアログの[閉じる] |
  | 親子 | [登録][更新][削除][行追加][行削除] |
  | 親子選択 | [登録][更新][削除][検索][クリア] |
  | 一括 | [検索][クリア][一括保存][行追加][行削除] |
  | 階層構造 | [ノード追加][変更][削除][新規] |
  | 選択ダイアログ | [選択][キャンセル][閉じる] |

- 画面遷移を伴う業務固有の操作のみを記載する。
- 自動採番項目がある場合は以下の形式で記載する。採番ルール以外の目的でスタンダードボタンを本章に記載してはならない。
  - 基本: 「[新規作成]：＜採番項目名＞ に `(自動採番)` を表示する。保存時にシステムが ＜採番ルール＞ で採番する。」
  - 一括: 「[行追加]：＜採番項目名＞ に `(自動採番)` を表示する。[一括保存] 時にシステムが ＜採番ルール＞ で採番する。」
- **Excelアップロード機能がある場合**、エラー処理方針は以下をデフォルト仕様とし、TBDとして挙げない。
  - 1件でもエラーがあれば全件差し戻す（部分登録は行わない）。
- 該当なしの場合は「特になし」と記載する。

---

### 未確定事項

- 業務要件に明記されていない仕様を推測・仮確定した場合は、必ずTBDとして本章に列挙する。「常識的に決まる」と判断して省略してはならない。
- 記載形式は箇条書きとし、論点ごとに1項目でまとめる。
- 各項目の直下に、AIが期待する【回答例】を1行で追記する。
- TBDが存在しない場合のみ「特になし」と記載する。
- **以下の条件に該当する場合は必ずTBDに追加すること。**
  - コード・名称など一意性が必要そうな項目：「＜項目名＞は重複不可か」
  - 入力項目が合計30件を超える場合：「表示形態（ダイアログ / ドロワー）の確認」
  - **【親子・親子選択のみ】** 子テーブルが3つ以上ある場合：「表示形態（ダイアログ / ドロワー）の確認」
