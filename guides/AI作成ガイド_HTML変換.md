# AI作成ガイド HTML変換

## 目的

プロジェクトフォルダ内の全 Markdown 成果品（`.md`）を HTML に一括変換し、ブラウザで閲覧・共有できる形式を生成する。変換前にプロジェクトの Index ページも生成する。

---

## ワークフロー概要

| コマンド | 処理内容 |
|---------|----------|
| `HTML変換して` | Step A → Step B を順に実行 |

---

## Step A: Index ページ生成

`guides/AI作成ガイド_HTML変換.md`（このガイド）を参照して、プロジェクトフォルダ内の全成果品 Markdown ファイルを収集し、`index.md` を生成する。

**実行トリガー：**  
ユーザーが `HTML変換して` と依頼したら、カレントのプロジェクトフォルダを対象に Step A を実行する。

**出力先：** プロジェクトフォルダの直下（`projects/<グループ>/<業務名>/index.md`）

**出力ルール：**

- ファイル名は `index.md` 固定。既存ファイルがあれば上書きする。
- frontmatter は以下の形式で記述する。

  ```text
  ---
  id: "doc-style"
  toc:
    depth_from: 1
    depth_to: 3
  html:
    toc: true
  ---
  @import "/assets/doc-style.less"
  ```

- H1 タイトルは `# <業務名> 成果品一覧` とする。
- 各成果品セクションを `## <カテゴリ名>` で区切り、リンクを箇条書きで記載する。
- カテゴリは以下の3グループに分類し、該当ファイルが存在するカテゴリのみ出力する。

  | カテゴリ（## 見出し） | 含むファイル | 対象ファイルパターン |
  |---------------------|------------|-------------------|
  | 要件 | 業務要件 | `*_業務要件.md` |
  |      | 業務仕様書 | `*_業務仕様書.md` |
  |      | 外部仕様書 | `*_外部仕様書.md`、`*_外部仕様書_*.md` |
  | 設計 | UI仕様書 | `*_UI仕様書_*.md` |
  |      | データ定義書 | `*_データ定義書.md`、`*_データ仕様書.md` |
  | 実装 | 実装定義書 | `実装定義書/実装定義書_*.md` |

- `_extracted.md` は収集対象外とする。
- 存在しないカテゴリのセクションは省略する。

**テンプレート（出力例）**

```text
---
id: "doc-style"
toc:
  depth_from: 1
  depth_to: 3
html:
  toc: true
---
@import "/assets/doc-style.less"

# ＜業務名＞ 成果品一覧

## 要件

- [業務要件](＜業務名＞_業務要件.md)
- [業務仕様書](＜業務名＞_業務仕様書.md)
- [外部仕様書](＜業務名＞_外部仕様書.md)

## 設計

- [UI仕様書\_＜画面ベース名1＞](＜業務名＞_UI仕様書_＜画面ベース名1＞.md)
- [UI仕様書\_＜画面ベース名2＞](＜業務名＞_UI仕様書_＜画面ベース名2＞.md)
- [データ定義書](＜業務名＞_データ定義書.md)

## 実装

- [実装定義書\_＜画面ベース名1＞\_一覧](実装定義書/実装定義書_＜画面ベース名1＞_一覧.md)
- [実装定義書\_＜画面ベース名1＞\_入力](実装定義書/実装定義書_＜画面ベース名1＞_入力.md)
```

---

## Step B: Markdown → HTML 一括変換

index.md を含むプロジェクトフォルダ内の全 Markdown ファイルを HTML に一括変換する。

**実行コマンド：**

```powershell
node scripts/convert-to-html.js projects/<グループ>/<業務名>
```

- `<グループ>/<業務名>` は対象プロジェクトのフォルダパスに置き換える。
- 変換結果は `dist/` 配下に出力される（`dist/<グループ>/<業務名>/`）。
- `_extracted.md` は変換対象外。
- 既存の HTML は上書きされる。

**出力確認：**

変換完了後、`dist/<グループ>/<業務名>/index.html` をブラウザで開いて各成果品にアクセスできることを確認する。
