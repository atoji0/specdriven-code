# 仕様駆動開発

Excel から業務要件・業務仕様書・外部仕様書・UI仕様書・データ定義書を AI で自動生成し、そのままソースコードを生成するツール。

---

## 1. 全体の流れ

業務要件を記載した Excel を用意し、Copilot Chat に指示を出すだけで、業務仕様書 → 外部仕様書 → UI仕様書 → ソースコードの順に成果物が生成されます。

各ステップで仕様が未確定な箇所は **未確定事項（以下、TBD）** として自動的にドキュメントの「未確定事項」章へ記録されます。

- TBD が残っている間は次のステップへ進まず停止します
- TBD を確認・回答してから再実行してください
- 追加要望・質問・相談がある場合も TBD に追記してください
- 仕様書の本文を直接書き換えても構いませんが、**AI は TBD をもとに判断するため、書き換えた内容は必ず TBD にも追記してください**（本文だけでは AI は認識できません）

```mermaid
flowchart LR
    A[業務要件\n（Excel）] --> B[業務仕様書]
    B --> C[外部仕様書]
    C --> D[UI仕様書\nデータ定義書]
    D -. 任意 .-> E[整合性チェック]
    D --> F[ソースコード\n（Vue.js）]
    E --> F
```

## 2. GitHub Copilot の利用について

### 2.1 Premium Request の残量確認

Premium Request とは、高性能モデル（Claude Sonnet・Opus など）を利用する際に消費される**月ごとの利用枠**です。モデルによって消費量が異なり（後述の倍率を参照）、**毎月1日にリセット**されます。

VS Code の画面右下にある GitHub Copilot のアイコンをクリックすると、当月の使用率（%）を確認できます。

| プラン | 月間上限 | 利用制限 |
|--------|:--------:|---------------|
| **Business** | 300回 | 最新モデルが利用できない。(Claude Sonnet 4.6など) |
| **Enterprise** | 1,000回 | 特になし |

> [!TIP]
> 利用枠を節約するため、依頼内容はチャットに直接入力するのではなく、できる限りTBDに記述してからチャットで実行するよう心がけてください。

### 2.2 モデルの使い分け

| モデル | 倍率 | 特性 | 本ツールでの利用場面 |
|--------|:----:|------|---------------------|
| **GPT-4o** | 0x | 画像入力対応・軽量・高速 | 簡単な質問・確認、チャットに添付した画像の内容を解釈 |
| **Claude Sonnet** | 1x | 長文ドキュメントの構造的生成・複雑な推論 | 業務仕様書・UI仕様書・データ定義書・ソース生成（**推奨**） |
| **Claude Opus** | 3x | 最高水準の推論（高コスト） | 本ツールの用途では Sonnet で十分なため **不要** |

### 2.3 コンテキストの考え方

AI はチャット内の情報（コンテキスト）しか参照できません。
チャットが長くなると、古いやり取りが失われるため、チャット欄の右下にある〇ゲージが赤くなった場合、新しいチャットを始めてください。

## 3. 初回セットアップ

### 3.1 Git for Windows のインストール
[Git for Windows](https://gitforwindows.org/) をダウンロードしてインストールします。オプションはすべてデフォルトのままで構いません。

### 3.2 リポジトリのフォークとクローン

自分専用のコピーを GitHub 上に作成（フォーク）してからクローンします。こうすることで自由に修正でき、元のリポジトリの更新も後から取り込めます。

**フォーク（GitHub 上での操作）**
1. [specdriven-devkit リポジトリ](https://github.com/v097053_mazda/specdriven-devkit) をブラウザで開きます。
2. 右上の「Fork」のプルダウンボタンをクリックします。
3. 「Create a new fork」をクリックすると、自分のアカウント配下にコピーが作成されます。

**クローン（VS Code での操作）**
1. VS Code を起動し、左側の「ソース管理」アイコン（分岐マーク）をクリックします。
2. 「リポジトリのクローン」をクリックします。
3. 「GitHub からクローン」を選択すると、自分のリポジトリ一覧が表示されます。
4. `specdriven-devkit` を選択し、保存先フォルダを指定します。
5. 完了後「リポジトリを開く」をクリックして VS Code で開きます。

### 3.3 推奨拡張機能のインストール

1. 左側の「拡張機能」アイコン（四角が4つ並んだマーク）をクリックします。
2. 検索欄に `@recommended` と入力します。
3. 「ワークスペースのおすすめ」に表示された拡張機能をすべてインストールします。
   - GitHub Copilot Chat
   - Markdown All in One
   - Markdown Preview Enhanced
   - Japanese Language Pack for VS Code

### 3.4 Java のインストール（PlantUML 用）

PlantUML（図のレンダリング）には Java が必要です。以下の手順でインストールしてください。

1. 社内ポータルの [ソフトウェア ダウンロードページ](http://www.isd.mazda.co.jp/its/shr/use/desktoptool/download/index.html) を開きます。
2. 一覧から **Java（JRE）** を探してダウンロードします。
3. インストーラーを実行し、デフォルト設定のままインストールを完了させます。
4. インストール後、VS Code を再起動します。

> [!NOTE]
> Java がインストールされていない場合、Markdown Preview Enhanced で PlantUML の図がレンダリングされません。

## 4. VS Code の基本操作

### 4.1 フォルダを開く

メニューの「**ファイル**」→「**フォルダーを開く**」を選択し、クローンしたフォルダを指定します。

---

### 4.2 Markdown ファイルのプレビュー表示

1. 表示したい `.md` ファイルをクリックして開きます
2. エディター右上に表示される「プレビューを横に開く」アイコンをクリックします
3. 表示されたプレビューペインの右上にある `...` アイコンをクリックし、「グループをロック」を選択します

> [!TIP]
> グループをロックすることで、他のファイルを選択してもプレビューが右側に固定されたままになります。

---

### 4.3 チャットの操作

1. VS Code 上部のクイックアクセスバー横にある Copilot アイコンをクリックしてチャットパネルを開きます
2. 入力欄のモデル選択ドロップダウンで使用するモデルを切り替えます（上記「モデルの使い分け」を参照）
3. 参照させたいファイルをチャット欄にドラッグ＆ドロップして添付します
   - Excel ファイルは直接添付できないため、ファイルが入っているフォルダをドロップしてください
4. 依頼内容を入力して送信します（具体的なコマンド例は「使い方」章を参照）

---

### 4.4 変更を GitHub に保存する

生成した仕様書を GitHub に保存するには、VS Code のソース管理からコミット＆プッシュします。

1. 左側の「ソース管理」アイコンをクリックします。
2. コミットメッセージを入力して「**✓ コミット**」ボタンをクリックします。
3. 「**変更の同期**」ボタンをクリックして GitHub に反映します。

---

### 4.5 元リポジトリの更新を取り込む

元のリポジトリに修正が加わったときは、以下の手順で自分のフォークに取り込みます。

1. ブラウザで自分のフォーク（`https://github.com/<自分のアカウント>/specdriven-devkit`）を開きます。
2. 「Sync fork」ボタンをクリックします。
3. 「Update branch」をクリックして、フォークを最新の状態に同期します。
4. VS Code のソース管理アイコンをクリックし、「…」メニューから「プル」を実行してローカルにも反映します。

## 5. 使い方

### 5.1 プロジェクトの準備

`projects/` 配下に業務名のフォルダを作成し、業務要件を記載した Excel ファイルを配置します。

```
projects/
└── <業務名>/
    └── <業務名>_業務要件.xlsx
```

Excel ファイルには以下の内容を記載してください。AIはこの内容をもとに仕様書を生成します。

- **業務概要** — 現状の業務フロー・課題・改善要望をテキストで記述する
- **作りたい画面** — 画面名と目的を列挙する（例：「工事計画を一覧で編集したい」「ワークフローで申請したい」）
- **サンプルデータ** — 実際の運用に近いサンプルを数行記載すると、AI の生成精度が上がる
- **プルダウン候補** — 状態・種別・優先度などの選択肢一覧を記載する
- **用語集** — 業務固有の用語や略語があれば定義を記載する（例：モデル、AIC など）

---

### 5.2 業務仕様書を生成

Excel ファイルをもとに `*_業務要件_extracted.md` → `*_業務要件.md` → `*_業務仕様書.md` を自動生成します。  
TBD がなければそのまま外部仕様書の生成まで続けて実行します。

- Excel ファイルが存在するフォルダをチャットに追加して実行します。**Excel ファイルは直接添付できないため注意。**
```
仕様書を作成して
```

TBD がある場合は、`*_業務仕様書.md` の「未確定事項」章に追記・回答して保存してから実行します。
```
TBDに回答しました。続きをおねがい
```

---

### 5.3 外部仕様書を生成

`*_業務仕様書.md` をもとに外部仕様書（画面一覧・画面ワイヤー等）を生成します。

- `*_業務仕様書.md` をチャットに追加して実行します。
```
続きをおねがい
```

---

### 5.4 UI仕様書 ＋ データ定義書を生成

`*_外部仕様書.md` をもとに全画面分の UI仕様書を生成します。TBD がなければそのままデータ定義書まで自動生成します。

- `*_外部仕様書.md` をチャットに追加して実行します。
```
続きをおねがい
```

TBD がある場合は、該当の `*_UI仕様書_*.md` の「未確定事項」章に追記・回答して保存してから実行します。
```
TBDに回答しました。続きをおねがい
```

---

### 5.5 成果品の整合性チェック（任意）

業務仕様書・UI仕様書・データ定義書とガイドの整合性を検証し、形式不備の自動修正と未確定事項の追記を行います。

- プロジェクトフォルダをチャットに追加して実行します。
```
成果品をチェックして
```

---

### 5.6 ソースコードを生成

UI仕様書・データ定義書をもとに Vue + ElementPlus + IndexedDB のソースコードを生成します。Workflow 画面は出力対象外です。

> **Batch ごとに新しいチャットを作成してから「実装して」と依頼すること。**

- プロジェクトフォルダと対象の実装タスク書（`{業務名}_実装タスク_{N:02}.md`）をチャットに追加して実行します。
```
実装して
```

修正・追加依頼がある場合：

1. 実装タスク書（`{業務名}_実装タスク_{N:02}.md`）の `## 1. 確認事項・追加依頼` に内容を追記します
2. 「実装して」と依頼します
3. AI が対応後、該当箇所を削除します

```
│   ├── AI作成ガイド_業務仕様書.md                 # 業務要件.md → 業務仕様書.md の生成ルール
│   ├── AI作成ガイド_外部仕様書.md                 # 業務仕様書.md → 外部仕様書.md の生成ルール
│   ├── AI作成ガイド_UI仕様書.md                   # 通常・コンテキスト・参照・選択 UI仕様書の生成ルール
│   ├── AI作成ガイド_UI仕様書_Workflow.md          # Workflow UI仕様書の生成ルール
│   ├── AI作成ガイド_UI仕様書_帳票.md              # 帳票 UI仕様書の生成ルール
│   ├── AI作成ガイド_データ仕様書.md               # UI仕様書群 → データ定義書.md の生成ルール
│   ├── AI作成ガイド_実装タスク.md                 # 実装タスク書の生成ルール
│   └── impl/                                      # ソース生成用：実装ガイド群
│       ├── AI作成ガイド_実装_画面共通.md
│       ├── AI作成ガイド_実装_データモデル.md
│       ├── AI作成ガイド_実装_Repository.md
│       ├── AI作成ガイド_実装_Service.md
│       ├── AI作成ガイド_実装_API.md
│       ├── AI作成ガイド_実装_ルーティング.md
│       ├── AI作成ガイド_実装_一覧.md
│       ├── AI作成ガイド_実装_入力.md
│       ├── AI作成ガイド_実装_入力(一括).md
│       ├── AI作成ガイド_実装_選択.md
│       ├── AI作成ガイド_実装_Tabulator.md
│       └── AI作成ガイド_実装_Excel対応.md
├── scripts/
│   └── Read-ExcelToMarkdown.ps1                   # Excel をフラットなテキストに変換するスクリプト
├── mock/
│   ├── src/                                       # ソース生成先（Vue アプリ本体）
│   └── src_ref/                                   # 実装サンプル参照元
└── projects/
    └── <業務名>/
        ├── <業務名>_業務要件.xlsx                  # 入力元の Excel ファイル
        ├── <業務名>_業務要件_extracted.md          # AI 生成：Excel から抽出したフラットなテキスト
        ├── <業務名>_業務要件.md                    # AI 生成：業務要件
        ├── <業務名>_業務仕様書.md                  # AI 生成：業務仕様書（画面一覧・データ一覧等）
        ├── <業務名>_外部仕様書.md                  # AI 生成：外部仕様書（画面ワイヤー・画面説明等）
        ├── <業務名>_UI仕様書_<画面ベース名>.md     # AI 生成：画面ごとの UI 仕様書
        ├── <業務名>_データ定義書.md                # AI 生成：データ定義書（全 UI 仕様書確定後）
        ├── <業務名>_実装タスク_01.md               # AI 生成：実装タスク書（Batch 01）
        └── <業務名>_実装タスク_NN.md               # AI 生成：実装タスク書（Batch N）
```
