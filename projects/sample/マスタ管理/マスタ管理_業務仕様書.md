# マスタ管理 業務仕様書

## 目的

本書は、マスタ管理業務の骨格（関係者・業務フロー・ユースケース）を整理し、外部仕様書・UI仕様書・データ定義書の作成および関係者間の認識合わせに使用する。

---

## 対象範囲

- 国情報の検索・登録・修正・削除
- 顧客情報の検索・登録・修正・削除

---

## 関係者（ロール）

| 関係者（実在） | ロール（システム） | 主な責務 |
|----------------|-------------------|----------|
| システム管理者 | 管理者            | 国情報・顧客情報の登録・修正・削除を行う |

---

## 業務の流れ

### マスタ管理フロー

国情報・顧客情報のマスタを登録・修正・削除する。

#### 業務フロー

```plantuml
@startuml フロー1_マスタ管理
skinparam backgroundColor white
skinparam activityBorderColor #6c8ebf
skinparam activityBackgroundColor #ffffff

|管理者|
start

group 要選択 {
  :UC1:国マスタ管理; <<#D6EAF8>>
  :UC2:顧客マスタ管理; <<#D6EAF8>>
}

stop
| |
@enduml
```

#### UC構成図

```plantuml
@startuml UC図_マスタ管理フロー
left to right direction
actor 管理者

rectangle "国マスタ管理" {
  (UC1: 国マスタ管理)
}

rectangle "顧客マスタ管理" {
  (UC2: 顧客マスタ管理)
}

database "国情報"
database "顧客情報"

管理者 --> (UC1: 国マスタ管理)
管理者 --> (UC2: 顧客マスタ管理)
(UC1: 国マスタ管理) --> 国情報 : <<create/update/delete>>
(UC2: 顧客マスタ管理) --> 顧客情報 : <<create/update/delete>>
(UC2: 顧客マスタ管理) ..> 国情報 : <<read>>
@enduml
```
