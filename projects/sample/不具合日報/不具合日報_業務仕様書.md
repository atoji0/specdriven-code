# 不具合日報 業務仕様書

## 目的

本書は、不具合日報業務の関係者・業務フロー・ユースケースを整理し、関係者間の認識を揃えることを目的とする。

---

## 対象範囲

- 職場・車種の登録・編集
- 日報単位（日付・職場・シフト）での不具合の登録・編集（現場タブレットからの入力）
- 過去の不具合の検索・参照・ダウンロード
- シフト完了後の不具合日報のワークフロー申請・回覧

---

## 関係者（ロール）

| 関係者（実在） | ロール（システム） | 主な責務 |
| ------------ | ------------ | -------------------------------------------------------------- |
| 品質管理部 担当者 | 作業者 | 不具合を登録・編集し、ワークフロー申請を行う |
| 品質管理部 シフトリーダー | シフト責任者 | 職場・車種を管理し、不具合日報を参照・回覧する |
| 品質管理部 現場責任者 | 現場責任者 | 申請された不具合日報を回覧・承認する |

---

## 業務の流れ

### 不具合記録・申請

不具合の発生から日報申請・回覧承認までの一連の流れを管理する。現場タブレットから不具合を入力し、シフト責任者が承認する。

#### 業務フロー

~~~plantuml
@startuml フロー1_不具合記録・申請
skinparam backgroundColor white
skinparam activityBorderColor #6c8ebf
skinparam activityBackgroundColor #ffffff

|作業者|
start
:日付・職場・シフトを確定する;
:UC1:不具合登録; <<#D6EAF8>>
:シフトを完了する;
:UC2:不具合日報申請; <<#D6EAF8>>

|シフト責任者 / 現場責任者|
:UC3:日報回覧・承認; <<#D6EAF8>>
if (承認か？) then (承認)
  stop
else (差し戻し)
  |作業者|
  :内容を修正して再申請する;
  stop
endif

@enduml
~~~

#### 状態遷移図

不具合日報の申請から承認を経て完了に至るステータス遷移を表す。

~~~plantuml
@startuml 状態遷移_不具合日報
left to right direction

申請中 : 作業者
承認中 : シフト責任者 → 現場責任者
確認中 : 作業者
再申請中 : 作業者

[*] --> 申請中
申請中 --> 承認中 : 申請
承認中 --> 承認中 : 次ステップへ
承認中 --> 確認中 : 承認完了
確認中 --> [*] : 完了
承認中 --> 再申請中 : 差し戻し
再申請中 --> 承認中 : 再申請（ステップ1から）
再申請中 --> [*] : 取下げ
@enduml
~~~

**② 承認ステップ詳細表**

| ステップ | 承認者 | 次ステップ | 差し戻し先 |
|:-------:|--------|:--------:|----------|
| 1 | シフト責任者 | 2 | 再申請中（作業者） |
| 2 | 現場責任者 | 承認完了 | 再申請中（作業者） |
