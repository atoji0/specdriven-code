## 1. GitHub Copilot の利用について

### 1.1 Premium Request の残量確認

Premium Request とは、高性能モデル（Claude Sonnet・Opus など）を利用する際に消費される**月ごとの利用枠**です。モデルによって消費量が異なり（後述の倍率を参照）、**毎月1日にリセット**されます。

VS Code の画面右下にある GitHub Copilot のアイコンをクリックすると、当月の使用率（%）を確認できます。

| プラン | 月間上限 | 利用制限 |
|--------|:--------:|---------------|
| **Business** | 300回 | 最新モデルが利用できない。(Claude Sonnet 4.6など) |
| **Enterprise** | 1,000回 | 特になし |

> [!TIP]
> 利用枠を節約するため、依頼内容はチャットに直接入力するのではなく、できる限りTBDに記述してからチャットで実行するよう心がけてください。

### 1.2 モデルの使い分け

| モデル | 倍率 | 特性 | 本ツールでの利用場面 |
|--------|:----:|------|---------------------|
| **GPT-4o** | 0x | 画像入力対応・軽量・高速 | 簡単な質問・確認、チャットに添付した画像の内容を解釈 |
| **Claude Sonnet** | 1x | 長文ドキュメントの構造的生成・複雑な推論 | 業務仕様書・UI仕様書・データ定義書・実装定義書の生成（**推奨**） |
| **Claude Opus** | 3x | 最高水準の推論（高コスト） | 本ツールの用途では Sonnet で十分なため **不要** |

### 1.3 コンテキストの考え方

AI はチャット内の情報（コンテキスト）しか参照できません。
チャットが長くなると、古いやり取りが失われるため、チャット欄の右下にある〇ゲージが赤くなった場合、新しいチャットを始めてください。

## 2. 初回セットアップ

### 2.1 Git for Windows のインストールと初期設定

1. [Git for Windows](https://gitforwindows.org/) をダウンロードしてインストールします。オプションはすべてデフォルトのままで構いません。

2. インストール後、ターミナル（コマンドプロンプト または PowerShell）で以下を実行します。

   **プロキシ設定**（接続先ネットワークに応じていずれかを実行）

   ```bash
   # MHOCの場合
   git config --global http.proxy http://172.20.1.13:8080

   # マツダPCの場合
   git config --global http.proxy http://136.131.63.121:8082
   ```

   **ユーザー情報の設定**（コミット時に作成者として記録されます）

   ```bash
   git config --global user.email "yamada@example.com"
   git config --global user.name "Taro Yamada"
   ```

   > `yamada@example.com` と `Taro Yamada` の部分は自分のメールアドレスと氏名に書き換えてください。

3. 設定を確認します。

   ```bash
   git config --global --list
   ```

### 2.2 リポジトリのフォークとクローン

自分専用のコピーを GitHub 上に作成（フォーク）してからクローンします。こうすることで自由に修正でき、元のリポジトリの更新も後から取り込めます。

**フォーク（GitHub 上での操作）**
1. [specdriven-devkit リポジトリ](https://github.com/v097053_mazda/specdriven-devkit) をブラウザで開きます。
2. 右上の「Fork」のプルダウンボタンをクリックします。
3. 「Create a new fork」をクリックすると、自分のアカウント配下にコピーが作成されます。

**クローン（VS Code での操作）**
1. VS Code を起動し、左側の「ソース管理」アイコン（分岐マーク）をクリックします。
2. 「リポジトリのクローン」をクリックします。
3. 「GitHub からクローン」を選択すると、自分のリポジトリ一覧が表示されます。
4. `specdriven-devkit` を選択し、保存先フォルダを指定します。
5. 完了後「リポジトリを開く」をクリックして VS Code で開きます。

### 2.3 推奨拡張機能のインストール

1. 左側の「拡張機能」アイコン（四角が4つ並んだマーク）をクリックします。
2. 検索欄に `@recommended` と入力します。
3. 「ワークスペースのおすすめ」に表示された拡張機能をすべてインストールします。
   - GitHub Copilot Chat
   - Markdown All in One
   - Markdown Preview Enhanced
   - Japanese Language Pack for VS Code

### 2.4 Java のインストール（PlantUML 用）

PlantUML（図のレンダリング）には Java が必要です。以下の手順でインストールしてください。

1. 社内ポータルの [ソフトウェア ダウンロードページ](http://www.isd.mazda.co.jp/its/shr/use/desktoptool/download/index.html) を開きます。
2. 一覧から **Java（JRE）** を探してダウンロードします。
3. インストーラーを実行し、デフォルト設定のままインストールを完了させます。
4. インストール後、VS Code を再起動します。

> [!NOTE]
> Java がインストールされていない場合、Markdown Preview Enhanced で PlantUML の図がレンダリングされません。

### 2.5 Node.js のインストール（HTML 変換用）

Markdown 成果品を HTML 形式に変換する機能は Node.js を使用します。

1. [Node.js 公式サイト](https://nodejs.org/ja/) から **LTS 版** をダウンロードしてインストールします。

   > 社内ポータルからダウンロードできる場合はそちらを使用してください。

2. VS Code のターミナル（メニュー → 「ターミナル」 → 「新しいターミナル」）を開き、クローンしたフォルダに移動して以下を実行します。

   ```powershell
   npm install
   ```

   インストール完了後、`node_modules/` フォルダが生成されれば成功です。

> [!NOTE]
> `npm install` は初回セットアップ時のみ実行すること。以降は不要です。

## 3. VS Code の基本操作

### 3.1 フォルダを開く

メニューの「**ファイル**」→「**フォルダーを開く**」を選択し、クローンしたフォルダを指定します。

---

### 3.2 Markdown ファイルのプレビュー表示

1. 表示したい `.md` ファイルをクリックして開きます
2. エディター右上に表示される「プレビューを横に開く」アイコンをクリックします
3. 表示されたプレビューペインの右上にある `...` アイコンをクリックし、「グループをロック」を選択します

> [!TIP]
> グループをロックすることで、他のファイルを選択してもプレビューが右側に固定されたままになります。

---

### 3.3 チャットの操作

1. VS Code 上部のクイックアクセスバー横にある Copilot アイコンをクリックしてチャットパネルを開きます
2. 入力欄のモデル選択ドロップダウンで使用するモデルを切り替えます（上記「モデルの使い分け」を参照）
3. 参照させたいファイルをチャット欄にドラッグ＆ドロップして添付します
   - Excel ファイルは直接添付できないため、ファイルが入っているフォルダをドロップしてください
   - Excel のオートシェイプ等、スクリプトで読み取れないコンテンツが含まれる場合は、該当シートを `[PrtSc]` キーでスクリーンショットし（`ピクチャ/スクリーンショット/` に自動保存）、プロジェクトフォルダ直下に置いてチャットに添付してください（ファイル名は任意）
4. 依頼内容を入力して送信します

---

### 3.4 変更を GitHub に保存する

生成した仕様書を GitHub に保存するには、VS Code のソース管理からコミット＆プッシュします。

1. 左側の「ソース管理」アイコンをクリックします。
2. コミットメッセージを入力して「**✓ コミット**」ボタンをクリックします。
3. 「**変更の同期**」ボタンをクリックして GitHub に反映します。

---

### 3.5 元リポジトリの更新を取り込む

元のリポジトリに修正が加わったときは、以下の手順で自分のフォークに取り込みます。

1. ブラウザで自分のフォーク（`https://github.com/<自分のアカウント>/specdriven-devkit`）を開きます。
2. 「Sync fork」ボタンをクリックします。
3. 「Update branch」をクリックして、フォークを最新の状態に同期します。
4. VS Code のソース管理アイコンをクリックし、「…」メニューから「プル」を実行してローカルにも反映します。
